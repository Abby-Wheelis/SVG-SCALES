Scales!!!

As Murray quotes in the book, "scales are functions that map from an input domain to an output domain." Essentially, scales are functions used to take your data as an input and converts it to the pixels in order to scale down (or up) your dataset. Without scales, an svg that you are creating would have the data correspond directly to a pixel dimension, which would be highly inconcenient if your data point was say, 3 trillion dollars, it simply wouldn't be practical. Scales are useful in data visualizations becasue they allow you to convert the magnitude of your data into a corresponding pixel dimension in order to convey that magnitude in a clean visual. Additionally, scales are useful in data visualizations because they can be configured such that the visualization will respond well to added data, even if it was well outside the former domain or range. This is done with clever use of the d3.max() method.
